## 核心功能

### 1. 插件系统架构

```
插件 JSON (schema 2.0)
  ├─ source (必需)
  │   ├─ key
  │   ├─ display_name
  │   ├─ icon_name
  │   └─ permission_response_style
  ├─ detection (可选)
  │   ├─ process_names
  │   ├─ env_var_hints
  │   ├─ path_patterns
  │   └─ priority
  └─ hook_installation (可选)
      ├─ format
      ├─ config_path
      ├─ events
      ├─ timeout_seconds
      └─ extra_config (可选)
```

### 2. 检测系统

**自动检测流程**：
1. Bridge 收集进程族谱
2. PluginProcessDetector 匹配检测规则
3. SourceResolver 返回 CLI 源
4. 无需手动 --source 参数

**优先级**：
1. 显式 --source（最高）
2. 内置源（priority 1000+）
3. 插件源（priority 1-999）
4. Payload 检查（最低）

### 3. Hook 安装系统

**支持的格式**：
- **flat**: `[{event, command, timeout}]`（Cursor, Trae）
- **nested**: `{hooks: {Event: [{command, timeout}]}}`（Gemini, Codex）
- **claude-matcher**: `{hooks: {Event: [{matcher, hooks: [...]}]}}`（Claude）

**特性**：
- 路径扩展（~/ 和环境变量）
- 配置合并（保留用户现有条目）
- 幂等性（重复安装安全）
- Extra config 支持（Codex config.toml）

### 4. Bundled 插件

**已完成**：

## 技术架构

### 插件加载顺序

```
1. Bundled Plugins (bundled-plugins/)
   └─ 优先级：最高，用户无法覆盖

2. User Plugins (%AppData%/CodeIsland/sources/)
   └─ 优先级：次高，可自定义
```

### 检测引擎

```
PluginProcessDetector
  ├─ LoadDetectionRules()
  │   └─ 从所有插件提取检测规则
  ├─ SortByPriority()
  │   └─ 按 priority 降序排序
  └─ DetectFromAncestry()
      └─ 遍历规则，返回第一个匹配
```

### Hook 安装流程

```
ConfigInstaller.InstallPlugin(sourceKey)
  ├─ SourcePluginLoader.LoadPlugins()
  ├─ 查找插件 by sourceKey
  ├─ 提取 HookInstallationSpec
  ├─ HookStrategyFactory.Create(format)
  └─ strategy.Install(sourceKey, hookSpec)
      ├─ 读取现有配置
      ├─ 合并 CodeIsland 条目
      ├─ 写回配置文件
      └─ 处理 extra_config（如果存在）
```

## 安全特性

### PluginValidator

1. **路径安全**
   - 禁止绝对路径遍历（`../../../`）
   - 禁止 Windows 驱动器路径（`C:\`）
   - 允许 `~/` 和环境变量
   - 限制路径长度 ≤ 500 字符

2. **正则安全**
   - 禁止灾难性回溯模式
   - 限制嵌套深度 ≤ 3
   - 限制重复量词 ≤ 5
   - 限制正则长度 ≤ 200 字符

3. **资源限制**
   - 进程名 ≤ 50 个
   - 环境变量 ≤ 20 个
   - 路径模式 ≤ 10 个
   - 事件 ≤ 20 个
   - Timeout: 1-86400 秒
